# 🎓 ИИ-Бот «ВЫЖИВШИЙ СТУДЕНТ» (AI-Hub Multimode Bot)

Интерактивный многорежимный Telegram-бот, разработанный на базе фреймворка **Aiogram 3** и интегрированный с локальными и облачными языковыми моделями (**LM Studio / Groq API**). Проект представляет собой развлекательно-обучающую платформу с динамической сменой системных промптов и контекста «на лету».

---

## 🚀 Основной функционал и режимы

Бот поддерживает автоматическое управление историей диалога (до 15 сообщений на пользователя) и переключение между тремя уникальными режимами:

| Режим | Описание механики | Фишка |
| :--- | :--- | :--- |
| **😈 Вредный Профессор** | Интеллектуальный разбор кода на Python и математических задач. Бот сначала остроумно иронизирует над ошибками, а затем выдаёт идеальный академический разбор. | Тонкий юмор, строгая структура, обилие тематических эмодзи. |
| **💀 Попробуй Выживи: Экзамен** | Текстовый текстовый квест-симулятор. Профессор задаёт каверзные вопросы (варианты A, B, C, D). | Динамический расчёт **уровня стресса**. При достижении 100% — автоматическое отчисление. |
| **⏳ Учение-Мучение: Вне Времени** | Подменю со списком из 6 великих учёных. Бот интегрирует реальные исторические факты из Wikipedia. | Учёные забавно и философски обыгрывают современный молодёжный сленг. |

## 🛠 Архитектура проекта

Проект построен по модульному принципу, что исключает циклические импорты и упрощает масштабирование:

📂 `my_bot/`
 ├── 📄 `main_lmstudio.py`  # Главный исполняемый файл (для локального сервера)
 ├── 📄 `main_groq.py`      # Исполняемый файл для облачного Groq API
 ├── 📄 `prompts.py`        # Системные промпты (System Prompts) для ИИ и конфигурация ролей
 ├── 📄 `keyboards.py`      # Генерация клавиатур (ReplyKeyboardBuilder) под каждый режим
 ├── 📄 `wiki.py`           # Модуль интеграции с API Wikipedia для подгрузки реальных фактов
 ├── 📄 `.env`              # Файл конфигурации переменных окружения (Токены, Ключи)

## 💻 Стек технологий

* **Язык программирования:** Python 3.11+
* **Архитектура бота:** Асинхронный `Aiogram 3.x`
* **Работа с LLM:** Локальный сервер `LM Studio` (эмуляция OpenAI API, модель Qwen 3) / `Groq API` (Llama 3.3)
* **Инструменты:** `asyncio` для неблокирующих запросов, `requests`, `python-dotenv`.

---

## 🔧 Инструкция по установке и запуску

### 1. Клонирование и подготовка окружения
Убедитесь, что у вас установлен Python 3.11+. Установите необходимые зависимости:
```bash
pip install aiogram python-dotenv requests

```

### 2. Настройка переменных окружения

Создайте в корневой папке файл `.env` и добавьте ваши ключи:

```env
BOT_TOKEN="ВАШ_ТОКЕН_ОТ_BOTFATHER"
GROQ_API_KEY="ВАШ_КЛЮЧ_GROQ_ЕСЛИ_НУЖЕН"

```

### 3. Запуск через LM Studio (Локально)

1. Откройте приложение **LM Studio** и скачайте модель (например, `qwen3-1.7b`).
2. Перейдите во вкладку **Local Server** (иконка со стрелочками слева).
3. Нажмите зелёную кнопку **Start Server** (порт по умолчанию `1234`).
4. Запустите бота в терминале:

```bash
python main_lmstudio.py

```

## 📈 Особенности реализации на защиту (Для преподавателя)

1. **Асинхронность (Asynchronous I/O):** Все долгие HTTP-запросы к нейросети обёрнуты в `asyncio.to_thread`. Это предотвращает блокировку основного потока `aiogram`, бот может обрабатывать сообщения сотен пользователей одновременно без зависания клавиатур.
2. **State Management (In-Memory):** Состояние каждого пользователя (выбранный режим, уровень стресса, история сообщений) изолировано в словаре `user_states` по уникальному `user_id`. При смене режима память автоматически очищается во избежание «галлюцинаций» модели.
3. **Промпт-Инжиниринг:** Системные промпты используют строгую разметку структуры ответа (Few-shot/Zero-shot инструкции), заставляя модель выдерживать нужный тон и гарантированно возвращать техническое объяснение после шутливой части.
